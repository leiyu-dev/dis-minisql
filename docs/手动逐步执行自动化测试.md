# 手动逐步执行 scripts 自动化测试

本文档把 `dis-minisql/scripts/e2e_system_test.py` 的自动化端到端测试拆解为手动命令。按顺序执行后，可以覆盖脚本中的所有主要场景：启动 ZooKeeper、初始化分片元数据、启动三个 DataNode 和 Coordinator、建库建表、多数派写入、分布式查询、单节点故障、第 4 个节点扩容、rebalance、health 和 snapshot。

下面命令默认在单机本地执行，使用固定端口：

- ZooKeeper：`12181`
- Coordinator：`18080`
- `node-a`：`19001`
- `node-b`：`19002`
- `node-c`：`19003`
- `node-d`：`19004`

如果这些端口已被占用，请先修改本文中的端口变量和配置文件。

## 1. 准备构建产物

进入仓库根目录：

```bash
cd /home/ly527609/homework/dis-minisql
```

构建 MiniSQL C++ 主程序：

```bash
cmake -S minisql -B minisql/build
cmake --build minisql/build --target main -j2
```

构建 Java 分布式层：

```bash
mvn -q -f dis-minisql/pom.xml package
```

确认自动化测试依赖的三个关键文件存在：

```bash
test -x minisql/build/bin/main
test -f dis-minisql/target/dis-minisql-1.0.0.jar
test -x apache-zookeeper-3.9.5-bin/bin/zkServer.sh
```

如果上述任一命令没有输出且退出码为 `0`，说明对应文件存在；如果报错，需要先检查构建或 ZooKeeper 解压目录。

## 2. 创建手动测试运行目录

自动化脚本会用 `/tmp/dis-minisql-e2e-xxxxxx` 临时目录。手动测试建议使用固定目录，方便查看日志和清理：

```bash
cd /home/ly527609/homework/dis-minisql

export RUNTIME=/home/ly527609/homework/dis-minisql/tmp
export ZK_PORT=12181
export COORD_PORT=18080
export NODE_A_PORT=19001
export NODE_B_PORT=19002
export NODE_C_PORT=19003
export NODE_D_PORT=19004

rm -rf "$RUNTIME"
mkdir -p "$RUNTIME"/{logs,pids,zkdata,node-a,node-b,node-c,node-d}
```

后续命令依赖这些环境变量，请在同一个终端中继续执行。若换了终端，需要重新执行上面的 `export` 命令。

## 3. 生成 ZooKeeper 配置

自动化脚本会动态生成 `zoo.cfg`。手动执行时使用以下命令生成等价配置：

```bash
cat > "$RUNTIME/zoo.cfg" <<EOF
tickTime=2000
initLimit=10
syncLimit=5
dataDir=$RUNTIME/zkdata
clientPort=$ZK_PORT
admin.enableServer=false
EOF
```

检查配置：

```bash
sed -n '1,120p' "$RUNTIME/zoo.cfg"
```

应看到 `clientPort=12181` 和 `dataDir=/tmp/dis-minisql-manual-e2e/zkdata`。

## 5. 启动 ZooKeeper

后台启动 ZooKeeper，并把日志写到 `$RUNTIME/logs/zk.log`：

```bash
apache-zookeeper-3.9.5-bin/bin/zkServer.sh start-foreground "$RUNTIME/zoo.cfg" \
  > "$RUNTIME/logs/zk.log" 2>&1 &
echo $! > "$RUNTIME/pids/zk.pid"
```

等待端口打开：

```bash
python3 - <<'PY'
import os
import socket
import time

port = int(os.environ["ZK_PORT"])
deadline = time.time() + 20
while time.time() < deadline:
    try:
        with socket.create_connection(("127.0.0.1", port), timeout=0.5):
            print(f"ZooKeeper is ready on port {port}")
            break
    except OSError:
        time.sleep(0.2)
else:
    raise SystemExit(f"ZooKeeper port {port} did not open")
PY
```

如果失败，查看日志：

```bash
sed -n '1,200p' "$RUNTIME/logs/zk.log"
```

## 7. 启动三个初始 DataNode

仍在 `dis-minisql` 目录下执行：

```bash
java -jar target/dis-minisql-1.0.0.jar datanode "$RUNTIME/cluster3.json" node-a \
  > "$RUNTIME/logs/node-a.log" 2>&1 &
echo $! > "$RUNTIME/pids/node-a.pid"

java -jar target/dis-minisql-1.0.0.jar datanode "$RUNTIME/cluster3.json" node-b \
  > "$RUNTIME/logs/node-b.log" 2>&1 &
echo $! > "$RUNTIME/pids/node-b.pid"

java -jar target/dis-minisql-1.0.0.jar datanode "$RUNTIME/cluster3.json" node-c \
  > "$RUNTIME/logs/node-c.log" 2>&1 &
echo $! > "$RUNTIME/pids/node-c.pid"
```

等待并检查三个 DataNode 的 `/health`：

```bash
python3 - <<'PY'
import json
import os
import time
import urllib.request

ports = [
    int(os.environ["NODE_A_PORT"]),
    int(os.environ["NODE_B_PORT"]),
    int(os.environ["NODE_C_PORT"]),
]

for port in ports:
    deadline = time.time() + 25
    while time.time() < deadline:
        try:
            with urllib.request.urlopen(f"http://127.0.0.1:{port}/health", timeout=2) as response:
                body = json.loads(response.read().decode("utf-8"))
                print(f"DataNode on port {port} is ready: {body}")
                break
        except Exception:
            time.sleep(0.3)
    else:
        raise SystemExit(f"DataNode on port {port} is not ready")
PY
```

如果某个节点失败，查看对应日志：


## 9. 创建数据库和表

自动化脚本依次执行三条 DDL。手动执行时使用 `curl`：

```bash
curl -s -X POST "http://127.0.0.1:$COORD_PORT/sql" \
  -H 'Content-Type: application/json' \
  -d '{"sql":"create database dist;"}'
echo

curl -s -X POST "http://127.0.0.1:$COORD_PORT/sql" \
  -H 'Content-Type: application/json' \
  -d '{"sql":"create table t (id int unique, age int, primary key(id));"}'
echo

curl -s -X POST "http://127.0.0.1:$COORD_PORT/sql" \
  -H 'Content-Type: application/json' \
  -d '{"sql":"create table o (id int unique, user_id int, amount int, primary key(id));"}'
echo
```

通过条件：每条响应中都应包含 `"ok":true`。DDL 会广播到在线 DataNode，使各分片具备相同 schema。

## 10. 执行多数派写入测试

自动化脚本会插入三条用户数据和两条订单数据。手动执行：

```bash
for sql in \
  "insert into t values (1, 10);" \
  "insert into t values (2, 20);" \
  "insert into t values (3, 30);" \
  "insert into o values (101, 1, 7);" \
```

通过条件：

- 每条响应中包含 `"ok":true`。
- 写入响应的 `route` 字段中包含类似 `raft write` 的信息。
- 因为 `replicationFactor=3`，每个写入需要多数派副本确认才会成功。

## 22. 清理测试进程和临时目录

测试结束后，按相反顺序停止后台进程：

```bash
for name in coordinator node-d node-c node-b node-a zk; do
  pid_file="$RUNTIME/pids/$name.pid"
  if [ -f "$pid_file" ]; then
    pid="$(cat "$pid_file")"
    if kill -0 "$pid" 2>/dev/null; then
      kill "$pid"
    fi
  fi
done
```

等待进程退出：

```bash
sleep 2
```

如果仍有进程残留，可以强制终止：

```bash
for name in coordinator node-d node-c node-b node-a zk; do
  pid_file="$RUNTIME/pids/$name.pid"
  if [ -f "$pid_file" ]; then
    pid="$(cat "$pid_file")"
    if kill -0 "$pid" 2>/dev/null; then
      kill -9 "$pid"
    fi
  fi
done
```

确认端口已释放：

```bash
python3 - <<'PY'
import os
import socket

ports = [
    int(os.environ["ZK_PORT"]),
    int(os.environ["COORD_PORT"]),
    int(os.environ["NODE_A_PORT"]),
    int(os.environ["NODE_B_PORT"]),
    int(os.environ["NODE_C_PORT"]),
    int(os.environ["NODE_D_PORT"]),
]

for port in ports:
    sock = socket.socket()
    sock.settimeout(0.3)
    try:
        sock.connect(("127.0.0.1", port))
        print(f"port {port} is still open")
    except OSError:
        print(f"port {port} is closed")
    finally:
        sock.close()
PY
```

如果不再需要日志和数据，可以删除临时目录：

```bash
rm -rf "$RUNTIME"
```

## 23. 与自动化脚本的对应关系

| 自动化脚本动作 | 手动步骤 |
| --- | --- |
| 检查 jar、MiniSQL、ZooKeeper | 第 1 步 |
| 创建临时运行目录 | 第 2 步 |
| 生成 `zoo.cfg` | 第 3 步 |
| 生成 `cluster3.json` 和 `cluster4.json` | 第 4 步 |
| 启动 ZooKeeper | 第 5 步 |
| 执行 `init-zk` | 第 6 步 |
| 启动 `node-a`、`node-b`、`node-c` | 第 7 步 |
| 启动 Coordinator | 第 8 步 |
| 建库建表 | 第 9 步 |
| 多数派写入 | 第 10 步 |
| 聚合、排序、join | 第 11 至 13 步 |
| 停止 `node-a` | 第 14 步 |
| 宕机后继续写入 | 第 15 步 |
| 启动 `node-d` | 第 16 步 |
| 调用 `/admin/rebalance` | 第 17 步 |
| 调用 `/admin/health` | 第 18 步 |
| 调用 `/admin/snapshot` | 第 19 步 |
| 最终 `count(*)` 校验 | 第 20 步 |
| 自动清理进程和临时目录 | 第 22 步 |
